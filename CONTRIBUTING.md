# Contributing to Polylex

Polylex is in pre-alpha. The API surface is unstable and may change week to week. Contributions are welcome but coordinate first — file an issue before writing a PR larger than a one-line fix.

## Before you send a PR

1. **Sign the CLA.** The CLA bot will prompt you on your first PR. This lets the project potentially dual-license in the future (no content of your contribution changes; you always retain copyright).
2. **Keep PRs small.** Single-purpose, under ~300 lines of diff. Larger changes will be asked to split up.
3. **Write a test.** Anything touching cache logic, locale normalization, or network retry must come with unit tests.
4. **Match the style.** `./gradlew ktlintFormat` before committing.

## Code of conduct

Be kind. Assume good faith. Disagree on ideas, never on people.
